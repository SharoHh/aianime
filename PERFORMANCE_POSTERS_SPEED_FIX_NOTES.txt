AIanime performance + poster stability fix

Что изменено:
- Внешние постеры снова идут через /api/image по умолчанию, потому что cdn.myanimelist.net у части пользователей не открывается напрямую.
- /api/image теперь кеширует картинки на диске сервера, поэтому первый запрос может быть обычным, а повторные должны отдаваться быстро с VPS.
- Добавлены долгие Cache-Control заголовки для /api/image, /posters, /images и /_next/static.
- Supabase list read снова использует быстрый select только нужных колонок, но с fallback на select=* — каталог не должен падать на 40 seed-тайтлов.
- Увеличен runtime cache списков аниме до 5 минут.
- Главная тянет 180 тайтлов вместо 360.
- Страница тайтла сначала читает один тайтл по slug, и только потом меньший список похожих тайтлов.
- Из карточек убран тяжёлый episodesList массив, чтобы уменьшить payload между страницами.
- Для большинства картинок добавлены loading="lazy" и decoding="async".

После установки на VPS:
cd /var/www/aianime
git pull
npm install --registry=https://registry.npmjs.org/
rm -rf .next
npm run build
pm2 restart aianime --update-env

Проверка скорости:
curl -o /dev/null -s -w "home: %{time_total}\n" https://aianime.ru
curl -o /dev/null -s -w "catalog: %{time_total}\n" https://aianime.ru/catalog
curl -o /dev/null -s -w "catalog again: %{time_total}\n" https://aianime.ru/catalog

Проверка кеша постера:
curl -I "https://aianime.ru/api/image?url=https%3A%2F%2Fcdn.myanimelist.net%2Fimages%2Fanime%2F13%2F50521l.webp"
Первый раз может быть X-Aianime-Image-Cache: MISS, затем HIT.

Настройки:
AIANIME_IMAGE_PROXY_MODE=always  — стабильный режим, внешний CDN через /api/image. Рекомендуется.
AIANIME_IMAGE_PROXY_MODE=direct  — быстрый прямой CDN, но постеры могут не открываться у части пользователей.
AIANIME_LIST_CACHE_TTL_MS=300000 — кеш списков в памяти процесса.
AIANIME_IMAGE_CACHE_DIR=/tmp/aianime-image-cache — кеш картинок на диске.
